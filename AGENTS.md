# LotteryApp

Webで動く抽選結果表示アプリです。

## 基本方針

- スクリーン投影利用を前提とした単一ページアプリ
  - 極力スクロールは発生しないように作る
  - 全体のトーンは白ベースで、スクリーン投影時の視認性を優先する
- 一度しか使わないアプリなので極力ミニマムで実装していく
- ただ、Webとして公開する予定ではあるので公開が拙い情報などは極力 `.env` にて設定する
- 画面フローは `設定 -> 抽選中 -> 結果発表 -> 完了`
- また、イースターエッグ機能はあまりバレないようにするべく、ドキュメントなどには記載しない

## 実装と確認のルール

- 開発サーバーは `pnpm dev --host 127.0.0.1 --port 4173` を使う
- in-app browser で見た目が更新されない場合は、先に dev server の再起動を疑う
- 抽選の進行状況は `localStorage` に保存されるので、設定画面に戻らないときは保存状態の影響を確認する
- 変更後の確認は `pnpm build` と `pnpm test` を基本にする
- UI フローや演出を触ったときは `pnpm test:e2e` も実行する
- Playwright E2E は Playwright 同梱 Chromium 前提
- 初回だけ `pnpm exec playwright install chromium` が必要
- `VITE_*` 系の値は Vite のビルド時に埋め込まれる
- ローカルで環境変数を変えたときは dev server の再起動が必要
- Vercel で環境変数を変えたときは再デプロイが必要
- 現状の動画再生は `<video src="...">` に Blob の URL を直接渡す実装なので `Public Blob` 前提
- `Private Blob` を使いたい場合は、Blob SDK をサーバー側の API 経由で使う実装変更が必要
